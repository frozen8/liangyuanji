# 良缘纪 · 微信公众平台部署指南

> 本文档指导你完成「良缘纪」微信小程序从本地开发到上线发布的全流程。

---

## 一、环境准备

### 1.1 必备工具

| 工具 | 版本要求 | 用途 |
|------|---------|------|
| Node.js | ≥ 18.x | 编译构建 |
| npm | ≥ 9.x | 依赖管理 |
| 微信开发者工具 | 稳定版 ≥ 1.06 | 调试、上传、发布 |

### 1.2 账号准备

- **微信小程序账号**：在 [微信公众平台](https://mp.weixin.qq.com/) 注册（个人或企业）
- **AppID**：在小程序后台 → 「开发」→「开发管理」→「开发设置」获取
- **云开发环境**：在微信开发者工具中开通云开发，获取云环境 ID

---

## 二、项目配置

### 2.1 克隆项目

```bash
git clone <仓库地址>
cd 良缘纪
npm install
```

### 2.2 填入 AppID 和云环境 ID

#### 项目配置（project.config.json）

```json
{
  "appid": "你的AppID",
  "miniprogramRoot": "dist/",
  "cloudfunctionRoot": "cloudfunctions/"
}
```

#### 云开发初始化（src/app.tsx）

```typescript
Taro.cloud.init({
  env: '你的云环境ID',
  traceUser: true
});
```

> ⚠️ **重要**：AppID 和云环境 ID 必须匹配，否则云函数调用会失败。

---

## 三、本地编译

### 3.1 开发模式（热更新）

```bash
npm run dev:weapp
```

编译产物输出到 `dist/` 目录，修改源码会自动重新编译。

### 3.2 生产编译

```bash
npm run build:weapp
```

### 3.3 在微信开发者工具中调试

1. 打开微信开发者工具
2. 导入项目 → 选择项目根目录（非 dist 目录）
3. 工具会自动读取 `project.config.json`，识别 `miniprogramRoot: dist/`
4. 编译后即可在模拟器中预览

## 四、云函数部署

> ⚠️ **关键步骤**：云函数必须部署后小程序才能正常工作。

### 4.1 部署方法

在微信开发者工具中：

1. 左侧文件树找到 `cloudfunctions/` 目录
2. 右键点击需要部署的云函数文件夹
3. 选择 **「上传并部署：云端安装依赖」**
4. 等待上传完成（状态栏会显示进度）

### 4.2 需部署的云函数清单（23 个）

| 分类 | 云函数名 | 说明 |
|------|---------|------|
| 用户 | `login` | 登录 |
| 情侣 | `bindCouple` | 绑定 |
| 情侣 | `createCouple` | 创建关系 |
| 情侣 | `unbindCouple` | 解除绑定 |
| 情侣 | `getCoupleStatus` | 查询状态 |
| 情侣 | `updateWeddingDate` | 修改飞升日 |
| 灵兽 | `getBeast` | 获取灵兽 |
| 灵兽 | `feedBeast` | 番茄投喂 |
| 任务 | `getTasks` | 查询任务 |
| 任务 | `createTask` | 创建任务 |
| 任务 | `updateTask` | 更新任务 |
| 任务 | `deleteTask` | 删除任务 |
| 账本 | `getLedgers` | 查询账单 |
| 账本 | `createLedger` | 记账 |
| 账本 | `deleteLedger` | 删除账单 |
| 预算 | `getBudget` | 获取预算 |
| 预算 | `updateBudget` | 修改预算 |
| 修炼 | `startCultivate` | 开始修炼 |
| 修炼 | `finishCultivate` | 完成修炼 |

### 4.3 数据库集合初始化

在微信开发者工具 → 云开发控制台 → 数据库中创建以下集合：

```
users
couples
beasts
tasks
ledgers
budgets
cultivate_sessions
achievements
```

> 集合权限建议设置为「仅创建者可读写」或「所有用户可读，仅创建者可写」。

## 五、上传与发布

### 5.1 上传代码

1. 在微信开发者工具中点击右上角 **「上传」** 按钮
2. 填写版本号（如 `1.0.0`）和项目备注
3. 点击确定上传

### 5.2 体验版测试

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 「管理」→「版本管理」→「开发版本」
3. 找到刚上传的版本，点击 **「选为体验版」**
4. 在「成员管理」中添加体验者微信号
5. 体验者扫描体验版二维码即可测试

### 5.3 提交审核

1. 确认体验版功能正常
2. 在「版本管理」→「开发版本」中点击 **「提交审核」**
3. 填写审核信息（功能页面截图、测试账号等）
4. 等待微信团队审核（通常 1-3 个工作日）

### 5.4 正式发布

1. 审核通过后，在「版本管理」→「审核版本」中点击 **「发布」**
2. 小程序正式上线，所有用户可搜索使用

## 六、双账号测试流程

### 6.1 准备

- 两个微信号（A 方和 B 方）
- 两个微信开发者工具实例（或用同一台电脑切换账号）

### 6.2 绑定测试

1. A 方登录 → 仙府 → 邀请绑定 → 创建关系 → 获取邀请码
2. B 方登录 → 仙府 → 邀请绑定 → 输入邀请码 → 绑定
3. A 方自动检测到 B 方绑定，双方跳转首页

### 6.3 功能测试

| 功能 | 操作方 | 预期结果 |
|------|--------|---------|
| 发布任务 | A 方 | B 方降妖录中显示新任务 |
| 降服任务 | B 方 | 扣减灵石、获得修为、灵兽心情提升 |
| 记账 | A/B 方 | 双方灵石簿同步显示 |
| 修炼 | A/B 方 | 修为累加、番茄 +1、境界突破 |
| 投喂 | A/B 方 | 番茄 -1、饱食度 +20、心情 +10 |
| 解绑 | A 方发起 → B 方同意 | 双方回到未绑定状态，数据清空 |

## 七、常见问题

### Q1：云函数调用报错「user not found」

**原因**：用户未在 users 集合中创建记录。

**解决**：确保 `login` 云函数已部署，首次打开小程序会自动创建 user 文档。

### Q2：云函数报错「timeout」

**原因**：网络延迟或云函数冷启动。

**解决**：代码已配置 15 秒超时，偶发 timeout 属正常现象，重试即可。

### Q3：灵石簿余额不刷新

**原因**：页面缺少 useDidShow 刷新。

**解决**：已修复，从记账页返回后自动刷新。

### Q4：境界和灵兽形态未变化

**原因**：`finishCultivate` 或 `updateTask` 云函数未部署最新版本。

**解决**：重新部署这两个云函数（右键 → 上传并部署：云端安装依赖）。

### Q5：TabBar 图标不显示或切换异常

**原因**：自定义 TabBar 组件未正确加载。

**解决**：确认 `src/custom-tab-bar/` 目录存在，`app.config.ts` 中 `tabBar.custom: true`。
